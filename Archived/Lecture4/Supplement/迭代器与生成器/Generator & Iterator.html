1. 迭代 (Iteration)

迭代，又称遍历，指使用for循环对可迭代对象的每一个元素进行访问。

因此说到迭代，就和for循环相关。

for循环不仅可以作用于list、tuple，还可以作用于所有可迭代对象，例如字典等。

判断一个对象是否是可迭代对象：使用isinstance()函数判断是否是Iterable类型：

        
        1234
        from collections import Iterable

print(isintance('123', Iterable))   # True
print(isintance(123, Iterable))     # False

enumerate(可迭代对象)，将可迭代对象变成索引-值对，可以在for循环中同时迭代索引和元素本身。

3. 生成器 (Generator)

列表解析可以一次性直接创建一个列表。但因为内存限制，列表的容量有限，比如100万个元素的列表，怎么办？我们只需要其中的一些，其他的就浪费了？

若列表中的元素能够按照某种算法推算出来，那么可以在遍历的过程中，一个一个计算出后续的元素，而不是一次性全部计算出来。

一边循环，一边计算——生成器，generator。

1) 从列表解析到生成器

将列表解析的方括号换成小括号就可以了。

        
        123456
        # 列表解析，直接给到list
l = [x*x for x in range(1,11)]
print(l)    # [1, 4, 9, 16, 25, 36, 49, 64, 81, 100]
# 返回的是生成器
g = (x*x for x in range(1,11))
print(g)    # <generator object <genexpr> at 0x108ab2d58>

怎么访问生成器中的元素呢？说法不准确，生成器里并没有任何元素，它储存的是元素产生的算法 ，这也是为什么不能直接print出来。

使用next(g)一个一个打印元素，直到最后一个，再打印，报错StopIteration，一个生成器对象用完之后就废了。

但很少直接next，大多数时候使用for循环，因为generator是iterable的，可迭代对象。

2) yield 与生成器

遇到复杂的生成算法，可以使用yield配合函数来构造生成器。

Fibonacci数列：

        
        1234567
        def fib(max):
	n, a, b = 0, 0, 1
	while n < max:
		print(b)
		a, b = b, a+b
		n += 1
	return "done"

将上述代码中的print(b)改为yield b即使fib函数变成生成器，即：

        
        1234567891011
        def fib(max):
	n, a, b = 0, 0, 1
	while n < max:
		yield b
		a, b = b, a+b
		n += 1
	return "done"

f = fib(6)
print(f)
# <generator object fib at 0x10e59ddb0>

如果一个函数定义中包含yield关键字，那么这个函数就不再是一个普通函数，而是一个generator。

变为生成器的函数的执行顺序发生了改变：

  • 普通函数是顺序执行，遇到return语句，或者到了最后一行，返回None；
  • 变成generate的函数，每次调用next()的时候，执行到yield语句，返回生成的值，暂停；再次调用next()的时候，从上次返回的yield语句处继续执行。

想要得到最后return的值怎么办？使用except Exception as e捕获异常，直接print(e)或print(e.value)，将得到最后return的值。

        
        12345678
        def odd():
    print('step 1')
    yield 1
    print('step 2')
    yield(3)
    print('step 3')
    yield(5)
    return "Can you catch me?"

注意：要带参数，否则不是生成器，而是函数对象

        
        1234
        >>> odd
<function odd at 0x108852e18>
>>> odd()
<generator object odd at 0x108ab2ca8>

不能用for循环，需要用try/except来catch StopIteration，然后.value：

        
        1234567891011121314
        >>> a = odd()
>>> next(a)
step 1
1
>>> next(a)
step 2
3
>>> next(a)
step 3
5
>>> next(a)
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
StopIteration: Can you catch me?

捕获异常：

        
        1234567
        g = odd()
while True:
	try:
		next(g)
	except Exception as e:
		print(e.value)
		break

结果为：

        
        1234
        step 1
step 2
step 3
Can you catch me?

4. 迭代器 (Iterator)

迭代器：可以被next()函数调用并不断返回下一个值的对象，Iterator。

同样可以使用isinstance()判断一个对象是否是Iterator对象。

Generator因为可以使用next()，因此是Iterator。

但是List, Tuple, Str却不是Iterator，虽然他们是Iterable；但也可以使用iter()函数将其变成Iterator。

Iterator对象表示一个数据流，一个未来的有序序列，也就是当下不可能提前知道序列的长度，只有不断调用next()函数才能一个一个算出下一个元素，惰性的。

概念辨析：

  • 凡是可以被for循环的，都是Iterable；
  • 凡是可以作用于next()函数的，都是Iterator；
  • Iterable不一定是Iterator，但Iterator一定是Iterable的

包含关系：Generator⫋Iterator⫋IterableGenerator \subsetneqq Iterator \subsetneqq IterableGenerator⫋Iterator⫋Iterable
